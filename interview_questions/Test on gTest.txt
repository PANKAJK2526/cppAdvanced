Here’s a list of **medium-level Google Test (GTest) interview questions** focused on C++ unit testing:

---

### ✅ **GTest Basics**

1. What is the difference between `EXPECT_*` and `ASSERT_*` in GTest?
2. How do you write a basic test case in GTest?
3. What is the purpose of `TEST()` and how does it work?
4. How do you group related tests together in GTest?
5. How do you run only a specific test or test suite using command-line arguments?

---

### ✅ **Test Fixtures and Setup**

6. What is a test fixture in GTest?
7. What is the difference between `SetUp()` and `TearDown()` in test fixtures?
8. How do you define and use `TEST_F()` in GTest?
9. How do you share common setup and teardown logic between multiple test cases?
10. What are the differences between `TEST()` and `TEST_F()`?

### ✅ **Advanced Assertions**

14. How can you test for exceptions with custom matchers?
15. How can you test functions that return `void` but affect global or member state?

### ✅ **Parameterized and Typed Tests**
